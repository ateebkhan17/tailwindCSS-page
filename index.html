How I Installed Tailwind CSS

1. Execute these commands in your Terminal

> npm install tailwindcss
> npx tailwindcss init

2. Configure template paths in tailwind.config.js

File: tailwind.config.js

> content: ['./index.html'];

3. Create a CSS file and include Tailwind directives

File: tailwind.css

> @import 'tailwindcss/base';
> @import 'tailwindcss/components';
> @import 'tailwindcss/utilities';

4. Run the following command to apply TailwindCSS changes

> npx tailwindcss -i ./input.css -o ./output.css --watch

5. Include the generated output.css in your index.html

> link rel="stylesheet" href="output.css"

6. To automate changes, add this line to package.json

$ "scripts": { "build:css": "tailwindcss build styles.css -o output.css" },

How I Built the Page

I embarked on creating this webpage by starting with an empty directory and diligently following the installation steps. The preset Tailwind files seamlessly populated my directory, allowing me to craft the webpage using HTML, blending in Tailwind properties within the class names of the HTML tags. This process brought back memories of our earlier encounters with Bootstrap in this course.


The sequence involved the installation of Tailwind, generation of necessary files, and the seamless integration of Tailwind's utility classes directly into the HTML, eschewing the traditional approach of creating a separate CSS stylesheet.

Challenges

One formidable challenge surfaced during my coding journey with Tailwind CSS — finding the delicate equilibrium between customization and the utility-first approach. While Tailwind's utility classes offered a potent and efficient way to style elements, the quest for the perfect combination of classes for a specific design or layout proved to be a challenging endeavor.


Tailwind's extensive utility class library, although a blessing, introduced its own set of challenges. The plethora of options sometimes led to decision fatigue, making it arduous to select the most suitable classes for a particular styling requirement. This challenge prompted a thorough consideration of the trade-offs between customization and the simplicity that Tailwind advocates.


Furthermore, maintaining consistency in the application of styles across different components and pages demanded a disciplined approach to using Tailwind classes. Striking a balance between the convenience of utility classes and the desire for a consistent design language posed a challenge. Yet, this challenge spurred me to delve deeper into Tailwind's documentation, unveiling advanced features and best practices that could elevate and refine my workflow.
