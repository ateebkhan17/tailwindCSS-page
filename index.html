How I Installed Tailwind CSS

1. Execute these commands in your Terminal

> npm install tailwindcss
> npx tailwindcss init

2. Configure template paths in tailwind.config.js

File: tailwind.config.js

> content: ['./index.html'];

3. Create a CSS file and include Tailwind directives

File: tailwind.css

> @import 'tailwindcss/base';
> @import 'tailwindcss/components';
> @import 'tailwindcss/utilities';

4. Run the following command to apply TailwindCSS changes

> npx tailwindcss -i ./input.css -o ./output.css --watch

5. Include the generated output.css in your index.html

> link rel="stylesheet" href="output.css"

6. To automate changes, add this line to package.json

$ "scripts": { "build:css": "tailwindcss build styles.css -o output.css" },

Challenges and Solutions

One challenge I encountered using Tailwind CSS was: finding the balance between customization and the utility-first approach. While Tailwind's utility classes offered an efficient way to style elements, I had to dig deeper to find the perfect combination of classes for a specific design or layout that I wanted. This proved to to be a hard challenge at first. Tailwind's extensive utility class library, introduced its own set of challenges. The multitude of options sometimes led to decision fatigue, making it tiring to select the most suitable classes for a particular styling requirement. Furthermore, maintaining consistency in the application of styles across different components and pages demanded a disciplined approach to using Tailwind classes. Yet, this challenge allowed me to dive deeper into Tailwind's documentation, where I was able to find advanced features and best practices that I needed to solve these challenges. This exploration not only addressed the initial hurdles but also enhanced and refined my overall workflow with Tailwind CSS.
